Projekt terv — Google Classroom stílusú alkalmazás

1. Csoport struktúrája

  • Projektmenedzser: Felelős a feladatok kiosztásáért, a határidők követéséért és az esetleges akadályok elhárításáért. Rendszeres megbeszéléseket (pl. heti vagy napi rövid "stand-up") szervez.
  • Frontend fejlesztő: A felhasználói felület (UI) és a kliensoldali logika megvalósításáért felel.
  • Backend fejlesztő: A szerveroldali logika, adatbázis-kezelés és az API (Application Programming Interface) fejlesztéséért felel.
  • UI/UX tervező: A felhasználói élmény (UX) és a felület kinézetének (UI) megtervezéséért felel. Ha nincs külön ember, a frontend csapat veszi át ezt a feladatot.
  • Tesztelő / QA (Quality Assurance): A szoftver minőségének biztosításáért, a hibák felderítéséért és dokumentálásáért felel. Ezt a szerepet a csapattagok felváltva is betölthetik.

2. Fejlesztési terv: mérföldkövek, eredmények

  • 1. mérföldkő: Alapok és felhasználókezelés

    • Projekt inicializálása, verziókövetés (Git) beállítása.
    • Adatbázis séma megtervezése.
    • Felhasználói regisztráció, bejelentkezés és authentikáció (pl. JWT tokenekkel).
    • Alapvető felület (layout) létrehozása.

  • 2. mérföldkő: Kurzusok (tantermek) kezelése

    • Oktatók létrehozhatnak, módosíthatnak és törölhetnek kurzusokat.
    • Diákok csatlakozhatnak kurzusokhoz egyedi kód segítségével.
    • Felhasználói vezérlőpult, ahol a felhasználók látják a saját kurzusaikat.

  • 3. mérföldkő: Feladatok és bejegyzések létrehozása

    • Oktatók feladatokat és közleményeket tehetnek közzé a kurzus "falán".
    • A diákok láthatják ezeket a bejegyzéseket.
    • Fájlok csatolásának lehetősége a bejegyzésekhez.

  • 4. mérföldkő: Feladatok beadása és értékelése

    • Diákok beadhatják a megoldásaikat (szöveg, fájl).
    • Az oktatók megtekinthetik a beadott munkákat és értékelhetik (pontozhatják) azokat.

  • 5. mérföldkő: Befejező munkálatok

    • Felhasználói felület finomhangolása.
    • Átfogó tesztelés és hibajavítás.
    • Alkalmazás telepítése (deployment) éles környezetbe.

3. Programozói nyelvek, fejlesztő eszközök

  • Frontend (kliensoldal):

    • Keretrendszer: React
    • Nyelv: TypeScript
    • Stílusozás: Tailwind CSS és Styled Components

  • Backend (szerveroldal):

    • Környezet: Node.js
    • Keretrendszer: Express.js

  • Adatbázis:

    • Relációs (SQL): PostgreSQL

  • Fejlesztői eszközök:

    • Kódszerkesztő: Visual Studio Code.
    • Verziókezelés: Git és GitHub/GitLab.
    • API tesztelés: Postman vagy Insomnia.

4. Használni tervezett eszközök, technikák

  • Módszertan: Scrum, a feladatok követésére pedig Trello és GitHub Projects.
  • Architektúra: RESTful API
  • Authentikáció: JSON Web Tokens (JWT)
  • CI/CD (Continuous Integration/Continuous Deployment): GitHub Actions, manuális unit tesztek
  • Konténerizáció (opcionális): Docker

5. Leendő produktumok (dokumentum), és létrejöttük ideje

  • Projektterv (ez a dokumentum): A projekt elején.
  • Technikai dokumentáció: Folyamatosan a fejlesztés során. API végpontok leírása OpenAPI-al.
  • Forráskód: Egy GitHub repository-ban
  • Tesztelési dokumentáció: A tesztelési fázisban, amely tartalmazza a teszteseteket és az eredményeket.
  • Működő alkalmazás (demó): A projekt végén, egy publikusan elérhető linken.

6. Tesztelési követelmények

  • Funkcionális tesztelés
  • Usability (használhatósági) tesztelés
  • Kompatibilitási tesztelés
  • Teljesítménytesztelés
  • Biztonsági tesztelés

7. Bemutatás (demó) és szállítás módja, ideje

  • Szállítás:

    • A teljes forráskód feltöltése egy nyilvános GitHub repository-ba.
    • Egy részletes README.md fájl, amely leírja a projekt célját, a használt technológiákat és a telepítési lépéseket.
    • Link a működő, telepített alkalmazáshoz (pl. Vercel, Heroku, vagy más felhőszolgáltató segítségével).

  • Bemutatás:

    • Élő demó a félév végi prezentáción, ahol bemutatjátok a fő funkciókat oktatói és diák szerepkörben is.

  • Időpont: A projekt utolsó hetében, a leadási határidő előtt.

8. Információforrások

  • Hivatalos dokumentációk: A választott nyelvek és keretrendszerek (React, Node.js, PostgreSQL stb.) hivatalos weboldalai.
  • Inspiráció: Maga a Google Classroom, valamint UI/UX design oldalak, mint a Dribbble vagy a Behance.
  • Online kurzusok és tutorialok: FreeCodeCamp, The Odin Project, és különböző YouTube csatornák, amelyek teljes webalkalmazások készítését mutatják be.
  • Közösségi oldalak: Stack Overflow, Reddit (pl. r/webdev, r/reactjs) a felmerülő problémák gyors megoldásához.
